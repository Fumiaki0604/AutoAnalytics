## レポート作成のルール

各仮説には検証ステータスが付いています。ステータスに従って判定してください。
- `supported`  : SQL結果が得られた → データに基づいて支持/棄却を判断する
- `no_data`    : 結果0行 → 「このセグメントには該当データなし」として記述する
- `error`      : SQL実行失敗 → 「検証不可」として記述し、原因を推測する
- `no_sql`     : SQL未生成 → 定性的な考察のみ記述する
